# Wordle — 牛刀拆解

**路径：Fast Track** | **3 个 prompt** | **领域：Game → Puzzle**

> Wordle 是一个 5 字母猜词游戏，每天一题，6 次机会。它用极简设计创造了社交裂变。
> 选 Fast Track 因为规则简单、UI 单一，3 个 prompt 就够。

---

## Phase 1: 理解产品

### 我观察到的

- **UI 极简**：6 行 × 5 列的方块网格，一个虚拟键盘，一个输入框——没有菜单、没有设置、没有广告
- **反馈机制**：绿色 = 字母对位置对，黄色 = 字母对位置错，灰色 = 字母不在答案里
- **每日一题**：所有玩家同一天看到同一个答案，午夜重置
- **分享机制**：完成后生成一个 emoji 方块图（🟩🟨⬜），可复制粘贴到社交媒体
- **无注册、无账号、无 App**：纯网页，一个 URL，打开即玩

### 我推断的

- **灵魂是"一句话就能解释规则"**：不需要教程，不需要 onboarding，看别人玩一次就会了
- **社交裂变是有意设计的**：emoji 分享图是 Wordle 的增长引擎——它把游戏结果变成了社交货币
- **稀缺性是钩子**：每天只有一题，玩完只能等明天。这不是 bug，是 retention 策略
- **MDA 分析**：
  - **Mechanics**：猜词、颜色反馈、6 次限制
  - **Dynamics**：排除法推理、字母频率直觉、策略选择（先猜元音多的词 vs 高频辅音）
  - **Aesthetics**：挑战（Challenge）、发现（Discovery）、 fellowship（社交分享）

---

## Phase 2: 拆解

**构建顺序：** Foundation → 核心游戏逻辑 → 分享功能

**为什么这个顺序：**
1. Foundation 搭 UI 骨架（网格 + 键盘）
2. 核心游戏逻辑是产品本身
3. 分享是增长层，最后加

**Not-To-Do（全局）：**
- 不做每日一题的服务端逻辑（用本地日期 + 预设词表模拟）
- 不做多语言
- 不做统计面板（streak、win rate）
- 不做硬模式（Hard Mode）
- 不做 App / PWA

---

## Phase 3: Prompts

### Prompt 0: Foundation

> 做一个猜词游戏页面——5 个字母的英文单词，猜 6 次。纯 HTML + CSS + JS 单文件。
>
> UI 只有三个部分：
> 1. 6 行 × 5 列的方块网格——每行一次猜测，每个方块显示一个字母
> 2. 虚拟 QWERTY 键盘（3 行：QWERTYUIOP / ASDFGHJKL / ENTER ZXCVBNM ⌫）
> 3. 顶部标题"WORDLE"，居中
>
> 方块样式：深色背景、白色边框、大号白色字母居中。键盘按钮深灰色、圆角、白色字母。
> 字体用等宽或无衬线，整个页面最大宽度 500px 居中。
>
> 现在只做 UI 骨架，不做任何交互逻辑。键盘按钮点击后在当前行的方块中显示字母就行。
>
> Done when：页面渲染出网格和键盘，点击键盘字母在网格中显示，点击 ⌫ 删除最后一个字母。
> 不要加：颜色反馈、胜负判定、动画、分享功能。

**What this adds:** UI 骨架——网格 + 键盘 + 基础输入。
**Why this prompt works:** 先把"能看到的东西"做出来，再添"能玩的东西"。骨架正确后面才顺。
**Pro tip:** 单文件 HTML 游戏是 vibe coding 的甜区——不需要 build 工具、不需要部署，浏览器打开就是成品。

---

### Prompt 1: 核心游戏逻辑

> 给猜词游戏加上完整的游戏逻辑。
>
> 答案词表：在 JS 里硬编码一个数组，包含 100 个常见的 5 字母英文单词（APPLE, BRAVE, CRANE, DANCE, EAGLE...）。游戏启动时从词表中随机选一个作为答案。
>
> 核心规则：
> 1. 玩家输入 5 个字母后按 Enter 提交
> 2. 每个方块根据答案变色：
>    - **绿色**（正确位置）：字母在答案中且位置对
>    - **黄色**（错误位置）：字母在答案中但位置不对
>    - **灰色**（不在答案）：字母不在答案中
> 3. 同时更新键盘上对应字母的颜色（一旦变绿就保持绿色，黄色同理）
> 4. 6 次用完或猜对 → 游戏结束
>
> 重要细节——**重复字母处理**：
> - 如果答案有两个 E，猜测只有一个 E，那个 E 只能标记一个位置（优先标绿）
> - 如果猜测有两个 E 但答案只有一个 E，只有其中一个标黄/绿，另一个标灰
> - 这是 Wordle 最容易写错的逻辑
>
> 游戏结束：
> - 猜对：显示"You got it!" + 猜了几次
> - 6 次用完：显示答案
> - 显示后出现一个"Play Again"按钮
>
> Done when：输入 5 字母按 Enter 后方块正确变色（绿/黄/灰），重复字母处理正确，猜对/猜完 6 次有结束提示，键盘字母颜色同步更新。
> 不要加：分享功能、动画、每日一题、统计。

**What this adds:** 完整可玩的游戏——从"能看到"变成"能玩"。
**Why this prompt works:** 单独处理了重复字母边界情况，这是 Wordle clone 里最常见的 bug。提前说清楚比事后 debug 省很多轮。
**Pro tip:** 游戏逻辑的 prompt 里，边界情况比主流程更重要。主流程 AI 都会写对，重复字母处理 80% 会写错。

---

### Prompt 2: 动画 + 分享

> 最后打磨游戏体验，加上动画和分享功能。
>
> 翻转动画：
> - 每个方块提交后有一个翻转效果——先绕 Y 轴翻转 90°（隐藏字母），变背景色，再翻回来显示颜色
> - 每个方块延迟 0.3 秒依次翻转（从左到右，波浪效果）
> - CSS 3D transform 实现，不要用 JS animation library
>
> 弹跳动画：
> - 输入字母时方块有一个轻微的弹跳（scale 1.0 → 1.1 → 1.0，150ms）
> - 无效单词时整行左右抖动（translateX 左右摆动两次）
>
> 分享功能：
> - 游戏结束后在结果下方显示一个"Share"按钮
> - 点击后复制到剪贴板，内容格式：
>   ```
>   Wordle 2026-04-21
>   🟩⬜⬜🟨⬜
>   ⬜🟨⬜⬜🟩
>   🟩🟩🟩🟩🟩
>   ```
> - 每行对应一次猜测，用 🟩（绿）🟨（黄）⬜（灰）
> - 第一行是游戏日期，后面是结果
> - 复制后按钮文字变成"Copied!"，2 秒后恢复
>
> Done when：提交猜测后方块依次翻转并变色，输入字母有弹跳，无效单词有抖动，游戏结束后可复制 emoji 分享图。
> 不要改动前面的游戏逻辑和颜色计算。

**What this adds:** 游戏的"感觉"和社交裂变能力。
**Why this prompt works:** 分层——先修动画（视觉层），再加分享（增长层）。明确"不要改动前面逻辑"防止 AI 回退。
**Pro tip:** Wordle 的分享图是它成功的秘密。emoji 格式让它在任何平台（Twitter、微信、Slack）都能完美渲染，不需要图片 hosting。

## 构建顺序

```
Prompt 0: UI 骨架（网格 + 键盘）    ← 5 分钟
Prompt 1: 游戏逻辑（猜词 + 反馈）   ← 核心价值
Prompt 2: 动画 + 分享              ← 锦上添花
```

**替代路径：**
- **极速版**：只做 Prompt 0 + 1，跳过动画和分享。10 分钟出一个可玩的 Wordle
- **完整版**：加 Prompt 3 做统计面板（streak、win rate、猜测分布柱状图），加 Prompt 4 做 Hard Mode

**下一步：**
- 把随机词表换成按日期选择（实现"每日一题"）
- 加 localStorage 存储历史记录
- 或者改变灵魂——把 5 字母英文换成 emoji 猜谜、数字猜谜、中文成语，看规则怎么跟着变
